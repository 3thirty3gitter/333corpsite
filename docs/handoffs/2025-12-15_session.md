## Issues Addressed

### 1. Image 400 Error - ✅ FIXED
**Problem:** Next.js was blocking external images from Momentec product API responses because the image domains weren't configured.

**Solution:** Updated `next.config.ts` to allow images from:
- `*.momentecbrands.com` / `momentecbrands.com`
- `*.momentecbrands.ca` / `momentecbrands.ca`
- `*.cloudfront.net` (common CDN)
- `*.amazonaws.com` (AWS S3)
- `*.sinalite.com` / `sinalite.com`

**Commit:** `4d136f7` - "fix: add image domains for Momentec and SinaLite CDNs"

### 2. Momentec Import 500 Error - 🔴 IN PROGRESS
**Problem:** POST to `/api/suppliers/momentec/import` returns 500 Internal Server Error when clicking import on a product.

**Investigation Done:**
- Added detailed debug logging to the import route
- Fixed duplicate variable name conflict (`admin` declared twice)
- The search API works correctly and returns products
- The 500 happens when clicking the "Import" button

**Likely Cause:** Missing `SUPABASE_SERVICE_ROLE_KEY` environment variable in Vercel production.

**Files Modified:**
- `src/app/api/suppliers/momentec/import/route.ts` - Added extensive debug logging

**Next Steps:**
1. Check Vercel environment variables for `SUPABASE_SERVICE_ROLE_KEY`
2. If missing, add it from Supabase Dashboard → Settings → API → service_role key
3. Redeploy and test import again
4. Check Vercel function logs for the debug output to pinpoint exact failure
